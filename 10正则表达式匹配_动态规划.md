# 动态规划入门：从正则表达式匹配理解 DP

> 如果你从未接触过动态规划，这篇文章将带你从 0 到 1 理解它。

---

## 一、什么是动态规划？

先别管专业术语，从一个生活场景说起：

> 你要爬 10 级楼梯，每次只能走 1 步或 2 步。问：有多少种走法？

**暴力做法**：把所有走法一条条列出来数一遍 —— 走法很多，而且会重复算很多次。

**聪明做法**：
- 走到第 1 级：只能从第 0 级走 1 步上来 → 1 种走法
- 走到第 2 级：可以从第 1 级走 1 步，或从第 0 级走 2 步 → 2 种走法
- 走到第 3 级：可以从第 2 级走 1 步上来（2 种），或从第 1 级走 2 步上来（1 种）→ 2+1 = 3 种
- **发现规律**：走到第 n 级的走法 = 走到第 n-1 级的走法 + 走到第 n-2 级的走法

这就是动态规划的核心思想：**大问题拆成小问题，小问题的答案存起来，后面直接用，不用重复算。**

### 动态规划的四个关键词

| 关键词 | 含义 | 一句话 |
|--------|------|--------|
| **状态** | 我们要记录什么东西 | "走到第几级" |
| **初始值** | 最基础、不用算就知道的答案 | "第 0 级：1 种（不动）" |
| **状态转移** | 怎么从小问题推到大问题 | "dp[n] = dp[n-1] + dp[n-2]" |
| **最终答案** | 我们想要的结果 | "dp[10]" |

**记住这四步，所有动态规划都逃不出这个框架。**

## 二、题目：正则表达式匹配

### 题目描述

给一个字符串 `s` 和一个模式 `p`，判断 `p` 是否能**完全匹配** `s`。

模式 `p` 中两个特殊字符：

| 字符 | 含义 |
|------|------|
| `.` | 匹配任意**单个**字符 |
| `*` | 匹配**零个或多个**它前面的那个字符 |

### 示例

```
示例 1: s = "aa",   p = "a"    → false   （"a" 只能匹配一个 a，不够）
示例 2: s = "aa",   p = "a*"   → true    （a* 表示 a 可以出现 0,1,2,... 次）
示例 3: s = "ab",   p = ".*"   → true    （.* 表示任意字符可以出现任意次）
示例 4: s = "aab",  p = "c*a*b"→ true    （c出现0次 + a出现2次 + b出现1次 = "aab"）
```

---

## 三、为什么这道题要用动态规划？

### 暴力想法（递归）

我们逐个字符比对：

```
s = "aab"
p = "c*a*b"

先看 p 的第一个字符 'c'：
  → 但 c 后面跟了 *，所以是 "c*" 整体
  → c* 可以匹配 0 个 c → 跳过，继续看 s="aab", p="a*b"
  → a* 可以匹配 2 个 a → 消耗 s 的前两个 a → 继续看 s="b", p="b"
  → b 匹配 b → 匹配成功！
```

看起来简单，但 `*` 有多种选择（0次、1次、2次……），每个选择都是一条路，会有大量**重复计算**：

```
                    s="aab", p="c*a*b"
                   /            \
          匹配0个c           匹配1个c（不可能，跳过）
              |
        s="aab", p="a*b"
           /      \
      匹配0个a    匹配1个a
         |           |
    s="aab",p="b"  s="ab",p="a*b"  ← 又会拆出更多分支...
```

当 s 和 p 稍长一点，分支爆炸。很多子问题（比如 "s='b', p='b' 能否匹配"）会被反复计算。

**动态规划就是来消除这种重复计算的：算过一次就存起来，下次直接查表。**

---

## 四、动态规划解法（详细拆解）

### 第一步：定义状态

**问自己**：我需要什么信息来描述一个子问题？

**答**：s 的前 i 个字符，能不能被 p 的前 j 个字符匹配。

所以定义：

```
dp[i][j] = true/false
```

> **dp[i][j]** 表示：s 的前 **i** 个字符能否被 p 的前 **j** 个字符完全匹配

**注意：i 和 j 是长度，不是下标！**

```
s = "aab"
s 的前 0 个字符 = ""（空串）
s 的前 1 个字符 = "a"
s 的前 2 个字符 = "aa"
s 的前 3 个字符 = "aab"
```

#### 为什么用长度而不是下标？

因为要考虑**空串**的情况。s 的下标 0 对应 "a"，但我们需要表示空串时，对应的是"下标 -1"，这不存在。用长度 0 就自然表示了空串。

**数组大小**：`dp[m+1][n+1]`，m 是 s 长度，n 是 p 长度。多出来的第 0 行和第 0 列留给空串。

### 第二步：找初始值

**最基础的情况**：两个都是空串

```
dp[0][0] = true    ← 空串匹配空模式，当然成立
```

**第一行**：s 是空串，p 不是空串，能匹配吗？

```
p = "c*a*b"
j=1: p[0]='c'     → dp[0][1] = false   （"" 能不能被 "c" 匹配？不能）
j=2: p[1]='*'     → dp[0][2] = true    （"" 能不能被 "c*" 匹配？能！让 c 出现 0 次）
j=3: p[2]='a'     → dp[0][3] = false   （"" 能不能被 "c*a" 匹配？不能，a 必须出现）
j=4: p[3]='*'     → dp[0][4] = true    （"" 能不能被 "c*a*" 匹配？能！a 也出现 0 次）
j=5: p[4]='b'     → dp[0][5] = false   （"" 能不能被 "c*a*b" 匹配？不能，b 必须出现）
```

**规律**：空串只能被包含 `X*` 的模式匹配（让 X 出现 0 次，就等于没有）。

```javascript
// 代码实现：初始化第一行
dp[0][0] = true;
for (let j = 2; j <= n; j++) {
  if (p[j - 1] === '*') {
    dp[0][j] = dp[0][j - 2];  // 看去掉 X* 后能否匹配空串
  }
}
```

### 第三步：状态转移（最核心！）

现在假设我们已经知道了所有更小规模的答案，怎么算 `dp[i][j]`？

看 `p` 的第 j 个字符（即 `p[j-1]`），分两种情况：

---

#### 情况 1：`p[j-1]` 是普通字符或 `.`

这时候很简单，p 的最后一个字符只能匹配 s 的最后一个字符（一对一）：

```
s = "ab"    → 最后一个字符 s[i-1] = 'b'
p = "ab"    → 最后一个字符 p[j-1] = 'b'
```

判断：
1. **当前字符能匹配吗？** → `p[j-1] == s[i-1]` 或 `p[j-1] == '.'`
2. **前面能匹配吗？** → 查表 `dp[i-1][j-1]`

```
两个条件都满足 → dp[i][j] = true
有一个不满足  → dp[i][j] = false
```

```
        dp[i][j]
          ↑
    当前字符匹配上 && 前面也匹配上
          ↑              ↑
   p[j-1]==s[i-1]     dp[i-1][j-1]
   或 p[j-1]=='.'
```

---

#### 情况 2：`p[j-1]` 是 `*`（重点！）

`*` 不能单独存在，它和前面的字符绑在一起。比如 `a*` 或 `.*`。

我们把 `p[j-2]` 记作 X（X 是字母或 `.`），那么 `X*` 有两种选择：

##### 选择 A：让 `X*` 匹配 0 次 → 直接扔掉

```
s = "ab"       p = "c*a*"
                      ↑
                   让 'a*' 匹配 0 次 → 等价于 s="ab" 能否被 p="c*" 匹配
```

```
dp[i][j] = dp[i][j - 2]
              ↑        ↑
           s不动    去掉X*，p倒退2位
```

##### 选择 B：让 `X*` 匹配 1 次或更多次

前提是：**s 的当前字符能被 X 匹配**。

```
i=2            j=4
s = "aab"      p = "c*a*b"
      ↑               ↑
    s[i-1]          p[j-2]（X，即 'a'）
```

如果 `s[i-1]` 能被 X 匹配，那就**消耗掉 s 的这一个字符**，然后继续看：

```
dp[i][j] = dp[i - 1][j]
                ↑    ↑
            s少一个   p不动！
```

##### ⚡ 为什么 p 不动？（最容易困惑的地方！）

因为 `*` 可以**重复匹配**。p[j] 这个位置就是 `X*`，它还能继续匹配 s 的上一个字符。所以 **p 不退**，s 退一步，问："s 少一个字符后，还能被 `X*` 匹配吗？"

换个角度理解：`dp[i-1][j]` 已经包含了 `X*` 匹配 0 次、1 次、2 次……的所有可能性。我们只需要"多消耗一个 s 的字符"就行。

```
举例：s="aaa", p="a*"
  dp[3][2]: a* 匹配第 3 个 a → 看 dp[2][2]
  dp[2][2]: a* 匹配第 2 个 a → 看 dp[1][2]
  dp[1][2]: a* 匹配第 1 个 a → 看 dp[0][2]
  dp[0][2]: a* 匹配 0 个 a → T ← 初始值！

  一路都是 T → 所以 "aaa" 能被 "a*" 匹配 ✓
```

##### 两种选择：满足一个就行

```
dp[i][j] = 匹配0次 || 匹配多次
         = dp[i][j-2] || (当前字符能匹配X && dp[i-1][j])
```

---

### 状态转移方程总结

```
                    ┌  dp[i][j-2] || (匹配 && dp[i-1][j])   当 p[j-1] == '*'
dp[i][j] =          └  匹配 && dp[i-1][j-1]                  当 p[j-1] != '*'

其中 "匹配" 指的是：
  - p[j-1] == '.' 或 p[j-1] == s[i-1]           （不是 '*' 的情况）
  - p[j-2] == '.' 或 p[j-2] == s[i-1]           （是 '*' 的情况，看 '*' 前面的字符）
```

## 五、完整图解

以 `s = "aab"`, `p = "c*a*b"` 为例，画出整个 DP 表：

```
       j=0   j=1   j=2   j=3   j=4   j=5
       ""    "c"  "c*"  "c*a" "c*a*" "c*a*b"
    ┌──────┬─────┬─────┬─────┬──────┬───────┐
i=0 │  T   │  F  │  T  │  F  │   T  │   F   │  s=""
    ├──────┼─────┼─────┼─────┼──────┼───────┤
i=1 │  F   │  F  │  F  │  T  │   T  │   F   │  s="a"
    ├──────┼─────┼─────┼─────┼──────┼───────┤
i=2 │  F   │  F  │  F  │  F  │   T  │   F   │  s="aa"
    ├──────┼─────┼─────┼─────┼──────┼───────┤
i=3 │  F   │  F  │  F  │  F  │   F  │   T   │  s="aab"
    └──────┴─────┴─────┴─────┴──────┴───────┘
                                         ↑
                                  最终答案：T ✓
```

### 几个关键格子怎么来的？（逐步推导）

#### `dp[0][2]`（"" 能否被 "c*" 匹配？）

- `p[1] = '*'`，X = 'c'
- **匹配 0 次**：`dp[0][0] = T`
- **匹配多次**："" 没有字符，不触发
- **结果**：T ✅

#### `dp[1][3]`（"a" 能否被 "c*a" 匹配？）

- `p[2] = 'a'`，不是 `*`
- `s[0] = 'a'` == `p[2] = 'a'` → 当前字符匹配 ✓
- 看前面：`dp[0][2]`（"" 能否被 "c*" 匹配？）= T
- **结果**：T ✅（c* 出现 0 次 + a 匹配 a = "a"）

#### `dp[1][4]`（"a" 能否被 "c*a*" 匹配？）

- `p[3] = '*'`，X = `p[2] = 'a'`
- **匹配 0 次**：`dp[1][2]`（"a" 能否被 "c*" 匹配？）= F
- **匹配多次**：`s[0]='a'` == `p[2]='a'` → T && `dp[0][4]` = T && T = T
- **结果**：F || T = **T** ✅

#### `dp[3][5]`（"aab" 能否被 "c*a*b" 匹配？）→ 最终答案

- `p[4] = 'b'`，不是 `*`
- `s[2] = 'b'` == `p[4] = 'b'` → 当前字符匹配 ✓
- 看前面：`dp[2][4]`（"aa" 能否被 "c*a*" 匹配？）= T
- **结果**：T && T = **T** ✅

## 六、完整代码

```javascript
/**
 * 正则表达式匹配（动态规划）
 * @param {string} s - 输入字符串
 * @param {string} p - 模式字符串，支持 '.' 和 '*'
 * @return {boolean}
 */
function isMatch(s, p) {
  const m = s.length;
  const n = p.length;

  // 创建 (m+1) x (n+1) 的二维数组，初始化为 false
  const dp = Array.from({ length: m + 1 }, () => Array(n + 1).fill(false));

  // 1. 初始值：空串配空模式
  dp[0][0] = true;

  // 2. 初始化第一行：空串能否被 p 的前 j 个字符匹配
  for (let j = 2; j <= n; j++) {
    if (p[j - 1] === '*') {
      dp[0][j] = dp[0][j - 2];  // 让 X* 出现 0 次
    }
  }

  // 3. 状态转移：从小问题推到大问题
  for (let i = 1; i <= m; i++) {
    for (let j = 1; j <= n; j++) {
      if (p[j - 1] === '*') {
        // 情况 A：X* 匹配 0 次 → 直接丢弃 X*
        const matchZero = dp[i][j - 2];

        // 情况 B：X* 匹配 1+ 次 → s[i-1] 能被 X 匹配
        const charMatch = p[j - 2] === '.' || p[j - 2] === s[i - 1];
        const matchOneOrMore = charMatch && dp[i - 1][j];

        dp[i][j] = matchZero || matchOneOrMore;
      } else {
        // 普通字符或 '.'：当前字符匹配 + 前面也匹配
        const charMatch = p[j - 1] === '.' || p[j - 1] === s[i - 1];
        dp[i][j] = charMatch && dp[i - 1][j - 1];
      }
    }
  }

  // 4. 最终答案
  return dp[m][n];
}
```

### 测试

```javascript
console.log(isMatch('aa', 'a'));               // false
console.log(isMatch('aa', 'a*'));              // true
console.log(isMatch('ab', '.*'));              // true
console.log(isMatch('aab', 'c*a*b'));          // true
console.log(isMatch('mississippi', 'mis*is*p*.')); // false
console.log(isMatch('aaa', 'a*a'));            // true
```

## 七、复杂度分析

- **时间复杂度**：O(m × n)，因为要填一个 (m+1) × (n+1) 的表格，每个格子 O(1) 计算
- **空间复杂度**：O(m × n)，需要二维 DP 表

> 题目约束 m, n ≤ 20，所以最多 21×21 = 441 个格子，非常快。

## 八、动态规划的通用套路

做完这道题，你会发现**所有动态规划都遵循同一个流程**：

```
1. 定义状态        dp[i][j] 表示什么？
2. 确定初始值      dp[0][0] = ?  第一行 = ?  第一列 = ?
3. 推导状态转移    dp[i][j] 怎么从之前的状态推出来？
4. 找到最终答案    答案在哪个格子里？
```

遇到任何动态规划题目，按这个顺序思考：

1. **先想状态**：子问题是什么？需要什么维度？（一维？二维？）
2. **再想初始值**：最小的、不用算就知道答案的情况是什么？
3. **再想转移**：当前状态依赖哪些已知状态？怎么算？
4. **最后看答案**：我们要的是 `dp[n]` 还是 `dp[m][n]`？

每次计算 dp[i][j] 时，依赖的格子都已经在左上方（已计算过）
